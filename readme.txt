PRECISION RELATIVEMENT A L'EXPLOITATION DU CMS

Trois repertoires principaux sont à la racine du projet.
- L'un nommé db contient le script article.sql nécésaire à la création de la base de donnée.
  La base de donnée doit se nommer article.db, et doit être incluse dans ce repertoire.
- Les deux autres : static et templates contiennent respectivement des scripts et des templates.
 
Un fichier nommé config.txt est également incluse à la racine de projet. Il contient les données permettant de se faire authentifier dans le site.
Après avoir intégré les données personnelles dans le fichier,(config.txt), 
- aller à la fin de la page d'accueil puis 
- cliquer sur configurer ou taper directement dans barre d'adresse localhost:5000/config
- s'authentifier avec son nom utilisateur et son mot de passe
 
La documentation de l'API est disponsible à la route localhost:5000/doc
